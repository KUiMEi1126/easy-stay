# Easy Stay 酒店管理系统

Easy Stay 是一个专为酒店行业设计的数字化管理平台，集成了管理员后台、商户管理端与移动端展示。项目模拟了真实的在线旅游（OTA）业务流程，涵盖了从商户入驻、酒店信息审核到移动端展示的完整链路。

## 📱 项目架构

项目采用三端分离的设计模式，确保了管理效率与用户体验的平衡：

* **管理后台 (admin-client)**
供管理员与商户使用。管理员负责全站酒店的审核与上下线控制；商户负责自身酒店信息、房型、价格的实时维护。基于 React 18 与 Ant Design 开发。
* **移动端展示 (mobile-client)**
为最终用户提供流畅的酒店浏览与查询体验。采用 antd-mobile 进行 H5 适配，优化了触屏交互。
* **服务端 (server)**
基于 Node.js Express 框架。提供 RESTful API 接口，负责鉴权、数据持久化及酒店图片的静态资源托管。

## 🛠️ 技术栈说明

### 前端技术

* 核心框架：React 18
* 构建工具：Vite
* 路由管理：React Router 6
* UI 组件库：Ant Design (PC端) / antd-mobile (移动端)

### 后端技术

* 运行环境：Node.js
* Web 框架：Express
* 文件处理：Multer (用于酒店门面图上传)
* 存储方案：本地轻量化 JSON 数据库

---

## ✨ 核心特性

* **多角色权限系统**
通过 Private Route 路由拦截器实现登录守卫。基于 localStorage 动态识别管理员（Admin）与商户（Merchant）身份，展示不同的功能菜单。
* **自动化资源管理**
集成图片上传流，支持酒店实拍图的上传、重命名与服务器自动托管，前端通过静态地址实时预览。
* **移动端的良好适配**
系统完美集成了精准的地理位置定位、多维度的酒店分类筛选（如价格区间、星级、房型）以及高效的模糊关键字检索功能。

## 📂 目录结构

* **admin-client/**：管理端源代码
* **mobile-client/**：移动端用户界面源代码
* **server/**：Express 后端服务
* **routes/**：模块化业务路由 (用户/酒店/上传)
* **public/**：存放酒店图片等静态资源
* **db.json**：数据存储文件

## 🚀 快速启动

### 1. 环境准备

确保本地安装了 Node.js (推荐 v18 及以上版本)。

### 2. 启动后端 API

* 进入目录：`cd server`
* 安装依赖：`npm install`
* 运行服务：`node app.js` (默认端口: 3000)

### 3. 启动前端

* **管理后台**：进入 `admin-client` 目录，执行 `npm install` 后运行 `npm run dev`。
* **移动端**：进入 `mobile-client` 目录，执行 `npm install` 后运行 `npm run dev`。
