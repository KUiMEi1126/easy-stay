# Easy-Stay 酒店预订系统部署指南

## 📱 方案一：内网穿透（推荐新手，快速测试）

使用内网穿透工具将本地服务暴露到公网，无需购买服务器。

### 使用 Ngrok（免费）

1. **下载 Ngrok**
   - 访问 https://ngrok.com/
   - 注册账号并下载 Windows 版本
   - 解压到任意文件夹

2. **启动后端穿透**
   ```powershell
   # 在 ngrok 文件夹中运行
   .\ngrok http 3000
   ```
   
   会得到一个公网地址，例如：`https://abc123.ngrok.io`

3. **修改前端配置**
   
   修改 `mobile-client/src/services/api.js`：
   ```javascript
   const request = axios.create({
     baseURL: 'https://abc123.ngrok.io/api',  // 改成 ngrok 地址
     timeout: 5000,
   });
   ```

4. **启动前端穿透**
   ```powershell
   # 在另一个终端中运行
   .\ngrok http 5173  # 假设前端运行在 5173 端口
   ```
   
   会得到前端地址，例如：`https://xyz456.ngrok.io`

5. **在手机上访问**
   - 直接在手机浏览器打开前端地址即可

### 使用 Cpolar（国内，更稳定）

1. **下载 Cpolar**
   - 访问 https://www.cpolar.com/
   - 注册账号并下载客户端
   - 安装后登录

2. **创建隧道**
   - 后端隧道：本地端口 3000 → HTTP 协议
   - 前端隧道：本地端口 5173 → HTTP 协议
   
3. **启动隧道并修改前端配置**（同 Ngrok）

## 🖥️ 方案二：云服务器部署（推荐生产环境）

购买云服务器（阿里云/腾讯云/华为云），部署完整项目。

### 2.1 准备服务器

1. **购买云服务器**
   - 系统：Ubuntu 20.04 或 CentOS 7+
   - 配置：2核2G内存即可
   - 带宽：1-5Mbps
   - 安全组：开放 80, 443, 3000 端口

2. **安装必要软件**
   ```bash
   # 安装 Node.js
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs
   
   # 安装 Nginx
   sudo apt-get install -y nginx
   
   # 安装 PM2（进程管理）
   sudo npm install -g pm2
   ```

### 2.2 部署后端

1. **上传代码**
   ```bash
   # 在本地打包后端
   cd server
   zip -r server.zip .
   
   # 上传到服务器（使用 FTP 或 scp）
   scp server.zip root@你的服务器IP:/opt/
   
   # 在服务器解压
   ssh root@你的服务器IP
   cd /opt
   unzip server.zip
   mv server easy-stay-server
   cd easy-stay-server
   npm install
   ```

2. **使用 PM2 启动**
   ```bash
   pm2 start app.js --name easy-stay-api
   pm2 save
   pm2 startup
   ```

### 2.3 部署前端

1. **修改 API 地址**
   
   修改 `mobile-client/src/services/api.js`：
   ```javascript
   const request = axios.create({
     baseURL: 'http://你的服务器IP:3000/api',
     timeout: 5000,
   });
   ```

2. **本地构建**
   ```powershell
   cd mobile-client
   npm run build
   ```
   
   会生成 `dist` 文件夹

3. **上传到服务器**
   ```bash
   # 本地打包
   cd mobile-client/dist
   zip -r dist.zip .
   
   # 上传到服务器
   scp dist.zip root@你的服务器IP:/var/www/
   
   # 在服务器解压
   ssh root@你的服务器IP
   cd /var/www
   unzip dist.zip -d easy-stay
   ```

### 2.4 配置 Nginx

在服务器上创建 `/etc/nginx/sites-available/easy-stay`：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    # 前端静态文件
    location / {
        root /var/www/easy-stay;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://localhost:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 图片上传目录
    location /uploads/ {
        alias /opt/easy-stay-server/public/uploads/;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/easy-stay /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 2.5 配置域名（可选）

1. 购买域名（阿里云/腾讯云）
2. 添加 A 记录指向服务器 IP
3. 修改 Nginx 配置中的 `server_name`
4. 申请 SSL 证书（使用 Let's Encrypt 免费证书）：
   ```bash
   sudo apt-get install certbot python3-certbot-nginx
   sudo certbot --nginx -d 你的域名
   ```

## 📦 方案三：Docker 部署（推荐团队协作）

### 3.1 创建 Dockerfile

**后端 Dockerfile** (`server/Dockerfile`)：
```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --production
COPY . .
EXPOSE 3000
CMD ["node", "app.js"]
```

**前端 Dockerfile** (`mobile-client/Dockerfile`)：
```dockerfile
FROM node:18-alpine as build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 3.2 创建 docker-compose.yml

项目根目录创建：
```yaml
version: '3.8'

services:
  backend:
    build: ./server
    ports:
      - "3000:3000"
    volumes:
      - ./server/db.json:/app/db.json
      - ./server/public/uploads:/app/public/uploads
    restart: always

  frontend:
    build: ./mobile-client
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: always
```

### 3.3 部署

```bash
# 本地构建镜像
docker-compose build

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 🔧 本地测试手机访问（同一WiFi）

如果只是想在同一WiFi下测试：

1. **查看电脑IP**
   ```powershell
   ipconfig
   # 找到 IPv4 地址，如 192.168.1.100
   ```

2. **修改后端监听地址**
   
   `server/app.js` 最后一行已经是：
   ```javascript
   app.listen(PORT, '0.0.0.0', () => console.log(`server run on ${PORT}`));
   ```

3. **修改前端 API 地址**
   
   `mobile-client/src/services/api.js`：
   ```javascript
   const request = axios.create({
     baseURL: 'http://192.168.1.100:3000/api',  // 改成你的电脑IP
     timeout: 5000,
   });
   ```

4. **重新启动前端**
   ```powershell
   cd mobile-client
   npm run dev -- --host
   ```

5. **手机访问**
   - 确保手机连接同一WiFi
   - 访问：`http://192.168.1.100:5173`

## ⚠️ 注意事项

1. **防火墙设置**
   - Windows：关闭防火墙或添加端口例外
   - Linux：使用 `ufw allow 3000` 开放端口

2. **CORS 配置**
   - 后端已配置 `cors()`，允许跨域访问

3. **数据库文件**
   - `server/db.json` 是数据文件，部署时需要同步
   - 建议改用 MySQL/MongoDB 数据库

4. **图片上传**
   - 确保 `server/public/uploads` 目录有写权限
   - 建议使用云存储（阿里云 OSS/腾讯云 COS）

5. **环境变量**
   - 创建 `.env` 文件管理配置：
     ```
     PORT=3000
     NODE_ENV=production
     ```
